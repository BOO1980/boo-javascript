Variables

  • A Variable is nothing more than a named reference to a piece of data
  • Variables are case sensitive
  • A Variable can hold a string, a number, a boolean, an array, another object, or a function reference
  • When you define a variable using the var keyword means you can access these variables on a local scope
  • If a variable is defined without the var keyword then the variable will be a global variable (which should be avoided)
    • The reason why you should avoid global variables is because the application is part of a larger JavaScript application
    • A variable of the same name may be in use in another part of the code in another file - and can be overridden whatever data is originally contained
    • Another libraries script could override the variable if it was global
  • Variables in JavaScript are basically named buckets of data, a way of creating a reference to that data so you are able to access that data again and again
  • Variables can use variables to persist data from one process to another.
  • Setting the scope of the variable is important if you have global variables with the same name
  • A variable can hold different data types at different times depending on context
  • You do not need to declare the data type of the variable
  • Variables in JavaScript are most likely to change datatype without notice
  • JavaScript variables have a identifier, scope, and a specific data type.
  • Each variable has an identifier which is unique that is unique to the scope of use- the identifier consists of a combination of letters, digits, underscores and dollar signs
  • Start variables with a letter, an underscore or a dollar sign
  • Variables cannot contain spaces or start with a number
  • The equals operator assigns a data value to a varaible
  • If there are any calculations to the right of the = operator those are performed before the value is assigned to the variable on the left of the operator
  • JavaScript has built in keywords, these keywords are off limits as variable names
  • After a value is assigned to a variable using the assignment operator, you can assign the value of that variable to another variable using the assignment operator.
  • When JavaScript variables are declared, they have an inital valye of undefined
  • If you do a mathematical operation on an undefined varaible the result will be NaN, which means "Not A Number"
  • If you concatenate a string with an undefined variable you will get a sting of undefined

The rules regarding scope

  • If you declare a variable with the var keyword in a function or block of code, its use is local to that function
  • if you use a variable without declaring its var keyword, and a global variable of the same name exists, the local variable is assumed to be the already existing global variable
  • If you declare a variable locally with a var keyword, but you do not initialise it (i.e. assigned a value) it is local and accessible but not defined
  • If you declare a variable locally without the var keyword, or explicitly declare it globally but do not initialise it, it is accessible globally but again not defined.
